# GoCore Changelog
Все изменения библиотеки GoCore будут документироваться на этой странице.

## 2026-07-23
### Added
- `mrlocale.Pool.LocalizerByCode()` - подбор локализатора по точному совпадению кода,
  без `language.Matcher` (`ru` при списке `["ru-RU"]` не совпадает). В отличие от `Localizer()`
  промах отличим от языка по умолчанию (`false`). Для источников, значения которых клиент
  получает от самого приложения (например, query-параметр смены языка);
- `util/timezone.LocationList.Default()` - пояс по умолчанию: первый загруженный пояс списка
  (симметрично `Bundle`); при пустом или негодном списке - `time.UTC`;

### Changed
- `util/timezone.LocationList.LocationByName()` стал строгим: имя принимается только при точном
  совпадении с поясом списка, а `Local` наружу больше не выдаётся (отвечает промахом). При промахе
  возвращается пояс по умолчанию (`Default()`) вместо `time.UTC`. Согласованно
  `util/timezone/config.ValidateTimeZones()` теперь тоже отвергает `Local` в списке.
  **ВНИМАНИЕ**: код, звавший `LocationByName("Local")` ради пояса процесса, теперь получит ошибку;

## 2026-07-20
### Added
- `mrlog/slog`: опция `WithTimeZone` и поле `TimeZone` в `wire/mrlog.LoggerConfig` —
  задают часовой пояс вывода времени: IANA-имя (например: `Europe/Moscow`), `UTC` или `Local`.
  Для IANA-имён требуется база часовых поясов в системе, в минимальных образах
  её нужно встроить импортом `_ "time/tzdata"`;
- `mrlog.ParseTimeZone` — преобразование имени часового пояса в `*time.Location`
  (хелпер логгера, рассчитан на однократный вызов при старте);
- `util/timezone` — `LocationList` с предзагруженными часовыми поясами и
  `config.ValidateTimeZones` для валидации их списка. Пояса загружаются один раз
  при старте, т.к. `time.LoadLocation` читает базу поясов с диска и работает медленно;
  `UTC` и `Local` доступны всегда. Это рекомендуемый способ работы с поясами
  в прикладном коде (например, для `mraccess.User.TimeZone()`).
  Проверка имён и создание списка разнесены: `ValidateTimeZones` отвергает
  пустые, повторяющиеся и отсутствующие в базе имена, а `NewLocationList`
  ошибку не возвращает и негодные имена молча пропускает. Поэтому валидация
  при загрузке конфигурации обязательна — иначе опечатка в имени пояса
  обнаружится не на старте, а на первом обращении к нему;
- `util/timezone`: `LocationList.NameByOffset` — подбор имени часового пояса
  по смещению относительно UTC и признаку летнего времени; при промахе
  сообщается `false`, чтобы вызывающий отличал ненайденный пояс от найденного.
  Индекс строится при создании списка сразу по всем состояниям, в которых пояс
  пребывает в течение следующего года, поэтому результат не зависит от времени
  года, в которое выполняется запрос. Часовой пояс процесса (`Local`) не подбирается
  никогда, так как вызывающему требуется IANA-имя; если пара подходит нескольким
  поясам (например, `Europe/Berlin` и `Europe/Paris` зимой), выигрывает
  последний указанный в списке, а пару `(+00:00, isDST=false)` всегда
  получает `UTC`, так как он регистрируется после всего списка;

### Changed
- `mrlocale.MessageProvider`: удалён метод `Languages() []language.Tag`.
  Список языков задаётся при создании `Bundle` и хранится им, поэтому спрашивать
  его у провайдера незачем: `Pool` подбирает локализатор по индексу, который
  возвращает `language.Matcher.Match`, а не по списку провайдера. Внешние
  реализации интерфейса метод могут удалить, наличие лишнего метода
  компиляции не мешает;
- `mraccess.User`: добавлен метод `TimeZone() string`, возвращающий IANA-имя
  часового пояса пользователя. Все внешние реализации интерфейса (моки в тестах,
  кастомные провайдеры) перестанут компилироваться, пока не будут дополнены этим методом;
- `mraccess.NewUser`: добавлен параметр `timeZone` (после `langCode`);
- `mrlog/slog`: время в логах теперь по умолчанию выводится в UTC во всех режимах
  (JSON, Text, цветной), ранее — в часовом поясе процесса. Чтобы вернуть прежнее
  поведение, задайте `TimeZone: "Local"`. Обратите внимание: форматы времени без
  смещения (`DateTime`, `TimeOnly`, `Kitchen`) не показывают, в каком поясе выведено время.
- `mrlog/slog`: в цветном режиме сообщение записи теперь раскрашивается (по умолчанию
  `color.White`), ранее выводилось цветом терминала. Цвет задаётся как у любого другого
  атрибута: `WithColorizeAttr("msg", colorKey, colorValue)`. Цвета назначаются по имени
  без учёта группы, поэтому этот же цвет получает и одноимённый атрибут, включая
  вложенный (`slog.Group("req", "msg", ...)`).
